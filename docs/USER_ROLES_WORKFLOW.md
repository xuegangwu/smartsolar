## 一、用户角色定义

### 1.1 角色矩阵

| 角色 | 代码 | 职责范围 | 典型操作 |
|------|------|----------|----------|
| 系统管理员 | `admin` | 系统配置、基础数据维护 | 电站录入、用户管理、备件库设置 |
| 运维主值 | `operator` | 监控告警、调度协调 | 盯盘、转工单、巡检调度 |
| 现场技术员 | `technician` | 现场作业执行 | 接单、填记录、拍照上传 |
| 值班长 | `supervisor` | 审核验收、紧急决策 | 工单验收、紧急情况升级 |
| 场站经理 | `manager` | 数据报表、运营分析 | 看KPI、导出报告 |

> **说明**：当前版本已实现 admin / operator / tech 三个内置账号。supervisor 和 manager 角色可在后续迭代中扩展。

## 二、人员体系（MongoDB 模型）

### 2.1 关联设计

```
┌─────────────────────────────────┐     1:1      ┌──────────────────┐
│  AuthUser (内存 / JWT)          │ ──────────→  │  Personnel        │
│  - username                      │              │  (MongoDB)         │
│  - role (admin/operator/tech)    │              │  - name, phone    │
│  - name                          │              │  - skills[]       │
└─────────────────────────────────┘              │  - stationId      │
                                                 │  - status         │
                                                 └──────────────────┘
```

**Personnel（人员档案）**：
- `_id`：人员唯一ID
- `authId`：关联的 AuthUser ID（一对一）
- `name`：姓名
- `phone`：联系电话
- `email`：邮箱
- `role`：角色（与 AuthUser.role 保持一致）
- `stationId`：所属电站（可为空，表示集团总部）
- `organization`：所属组织（总部 / XX区域公司）
- `skills`：技能标签 `["光伏", "储能", "充电桩"]`
- `status`：在职状态 `active` / `leave` / `fired`
- `joinDate`：入职日期

### 2.2 技术人员状态机

```
available ──→ busy ──→ available
    ↑___________|         |
    (处理完毕)              (新任务)
         │
         └──→ offline (下班/请假)
```

## 三、工作流闭环（完整链路）

### 3.1 告警 → 工单链路

```
┌─────────────┐
│  EMS 系统   │────── SSE 实时推送 ─────→  前端 Dashboard / 告警面板
│  (真实硬件) │                          🟢 实时显示
└─────────────┘

           告警触发（自动 / 手动）
                  │
                  ▼
        ┌──────────────────┐
        │   告警记录        │ ← 告警等级：critical / major / warning / info
        │   Alert Model    │
        └────────┬─────────┘
                 │ 一键"转工单"
                 ▼
        ┌──────────────────┐
        │   工单创建       │ ← 自动填充：标题/优先级/告警来源/描述
        │   WorkOrder      │
        └────────┬─────────┘
                 │ 派发
                 ▼
        ┌──────────────────┐
        │   派发给技术员   │ ← 选择执行人（从 Personnel 列表选）
        │   assigneeId     │
        └────────┬─────────┘
                 │
        ┌────────┴─────────┐
        ▼                   ▼
   【接受/拒绝】        【超时未处理】
        │                   │
        ▼                   ▼
   ┌─────────┐         ┌──────────┐
   │ 接单    │         │ 退回     │
   │accepted │         │ rejected  │
   └────┬────┘         └────┬─────┘
        │                   │
        ▼                   ▼
   ┌─────────┐         重新派发
   │处理中   │
   │processing│
   └────┬────┘
        │ 现场处理（填写处理记录）
        │ 可关联备件消耗
        ▼
   ┌──────────┐
   │ 验收中    │ ← 提交处理结果 + 照片
   │accepted_  │   通知值班长审核
   │check      │
   └────┬─────┘
        │ 审核通过 / 驳回
        ▼
   ┌─────────┐
   │ 已关闭   │ ← 完工归档
   │ closed   │
   └─────────┘
```

### 3.2 巡检计划链路

```
┌──────────────────────────────────────────────────┐
│                   巡检计划                       │
│  InspectionPlan (周期：daily/weekly/monthly)    │
└─────────────────────┬────────────────────────────┘
                      │ 自动生成 / 手动创建
                      ▼
             ┌──────────────────┐
             │ 巡检执行记录     │
             │ InspectionRecord │
             │ - 计划ID        │
             │ - 执行人        │
             │ - 执行时间      │
             │ - 结果(normal/异常待处理)│
             │ - 检查项详情    │
             └────────┬─────────┘
                      │ 结果为"异常待处理"
                      ▼
             ┌──────────────────┐
             │  自动创建抢修工单 │  type=fault, priority=urgent
             │  关联原计划      │
             └──────────────────┘
                      │
                      ▼
                  同"告警→工单"流程
```

### 3.3 备件消耗链路

```
工单关闭（closed）
      │
      ▼
┌────────────────────────────────┐
│ 自动扣减库存                   │
│ WorkOrder.spareParts[]        │
│  → SparePartConsume Record    │
│  → SparePart.quantity -= n    │
└────────────────────────────────┘
      │
      ▼ 库存 ≤ 安全库存
┌────────────────────────────────┐
│  自动触发低库存告警            │
│  Alert (level=warning)       │
└────────────────────────────────┘
```

## 六、消息通知体系

工单状态变更时，自动触发站内通知：

| 触发时机 | 通知对象 | 内容 |
|----------|----------|------|
| 工单创建 | 运维 + 技术员 | "新工单待派发：{title}" |
| 工单派发 | 被指定技术员 | "您有新的工单：{title}" |
| 技术员接单 | 运维主值 | "{name}已接单：{title}" |
| 技术员提交结果 | 值班长 | "工单待验收：{title}" |
| 工单验收通过 | 技术员 + 运维 | "工单已关闭：{title}" |
| 工单验收驳回 | 执行技术员 | "工单被驳回，请重新处理：{title}" |
| 备件库存不足 | 运维主值 | "备件库存不足：{name}（剩余{n}件）" |
| 巡检异常 | 运维主值 | "发现巡检异常：{planName}" |

> **当前状态**：Notification Model + API 已实现，触发点在各操作处分散嵌入。

  _id: ObjectId,

{
  ...
